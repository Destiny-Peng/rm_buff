# Buff (Kinematic & Prediction) 运动学与预测节点

本包（`buff`）是项目中最核心的后端模块，负责接收前端（视觉节点）解算出的装甲板2D/3D坐标，并对其进行时序上的三维运动学建模、预测与弹道补偿，最终生成云台绝对的 Pitch, Yaw 目标角度以进行击打。

## 模块架构

整个包按照功能可以划分为以下几个主要抽象层面：

1. **预处理（Preprocessor）**
   - 接收目标消息 `FanArmors`，完成时间对齐与坐标系变换（基于 `tf2_ros` 管理 `camera_link` 到 `base_link` 等坐标流转）。
2. **三维轨迹建模与优化（Modeling & Optimization）**
   - **RANSAC滤波**：剔除由于视觉误识别造成的跳变野值点。
   - **Ceres非线性优化**：通过 `ceres-solver` 进行三维空间圆形拟合（结合平面约束向量计算），估算出能量机关运动圆心的真实绝对空间位置、半径及法向量。
   - 这部分克服了单帧 PnP 解算深度抖动的问题，将离散的点集在时间尺度上平滑为三维空间中确定的运动圆盘。
3. **高阶弹道补偿（Ballistic Solver）**
   - 不同于常规的抛物线单向补偿，本模块实现了考虑**空气阻力**与**重力**的**四阶龙格-库塔（Runge-Kutta, RK4）**微分方程数值求壳。根据出膛初速、目标3D空间点、发射延时逼近最优击打姿态。
4. **开火控制器（Fire Controller）**
   - 订阅 `Readpack` 获取当前的云台实时位姿与系统时间戳。通过获取的时间结合发射机构延迟，动态调用运动模型请求未来击打点。
   - 判断预测点与当前系统的残差及运动状态，执行发弹控制与状态机跳转，并经 `Sendpack` 下发。

## 启动说明

提供了供无硬件测试的 Launch 启动脚本：

```bash
ros2 launch buff virtual_rune_launch.py
# 或应用于真车调试：
ros2 launch buff rune_launch.py
```

## 技术亮点（可作面试参考）
- **Ceres-Solver 三维圆约束拟合**：自定义 `ConstrainedCircle3DFittingCost` 的 CostFunction 约束空间多点，提升了系统对深度噪声的抗干扰能力。
- **RK4数值逼近求弹道**：将空气阻力作为速度的函数通过构建常微分方程解出迭代轨迹，相比传统的迭代近似或重力模型提升了预测准度。
