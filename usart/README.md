# USART (Serial Communication) 串口通信节点

本包（`usart`）主要功能是提供 ROS2 节点级别的硬件串口交互与数据解包打包能力，承担着“上下位机通信桥梁”的作用。在 Robomaster 的整个系统中，是系统命令下发的关键咽喉。

## 主要功能

本包定义了与下位机（如 STM32/其他 MCU）自定义通信协议，完成上下数据流的打包（`Pack`）处理与解包处理（`Unpack`）。主要包括：

1. **下发控制指令（`SendPack`）**
   - 接收来自视觉控制节点或 `buff` 节点的云台绝对 `pitch`、`yaw` 预测目标值，以及是否开火的控制命令。
   - 封包并加入固定包头、包尾以及校验位（CRC/或协议既定算法）发送至下位机。
2. **接收反馈姿态（`ReadPack`）**
   - 不断监听指定串口。解包来自 MCU 的信息。
   - 根据包类型（`RecvPackType` 等枚举）分离数据。常接收的包括：对时包（处理硬件与微型机系统间的时间戳误差）、当前位姿包（`roll`, `pitch`, `yaw` 和当前出膛初速等）。
   - 解析完成后转化为 `interfaces::msg::Readpack` 推送至前端或者运动学节点使用。
3. **硬件在环仿真（Virtual USART）**
   - 提供 `virtual_serialport` 用于脱离硬件底盘时，在本地或纯仿真环境中模拟串口接收回显发送内容，便于闭环验证 `buff` 包代码与视觉解算代码。

## 启动说明

提供了虚拟启动以供未接入下位机时调试：

```bash
# 启动虚拟串口测试模块
ros2 run usart virtual_serialport
# 启动真实串口节点（具体名称依编译目标而定）
ros2 run usart serialport
```

\* *注意: 若要运行真实物理串口节点，需要为目标串口（如 `/dev/ttyUSB0`）提供读写权限，建议配置 udev 规则。*
